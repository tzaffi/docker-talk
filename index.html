Docker - A Virtualization Technology

  • How's it different from Virtual Box or VMWare ?
    • Much lighter than an entire VM
    • Maintains encapsulation and isolation of full blown VM without overhead
    • Good example: run Python 2.7 and Python 3.2 apps on same Ubuntu box without worrying about ANYTHING (even installing Python)
    • Embraced by iPython / Jupyter community
    • Travis - Open Source CI: Speed up test spin-offs by moving from VM's to containers
    • DOWNSIDE: Security issues

Yodamemer Exercises

  1. Choose teams.
    At least 1 should have AWS access and 1 should know how to install and run a node.js project: Task Sheet http://bit.ly/1BCUBj1
  2. Run your task locally WITHOUT Docker (need node.js)
    1. Download the relevant files from task spreadsheet into new directory
    2. run npm install
    3. You'll need to put the MASHAPE KEY into the config.json
    4. See if you can figure out from your Dockerfile which command will get your app running
    5. See if you can figure out from your Dockerfile how to test your app
  3. Dockerize your application LOCALLY
    1. Build your docker image with
      docker build -t YOUR_IMAGE_NAME .
    2. run your docker container with
      docker run -p YOUR_LOCAL_PORT:8000 -d YOUR_IMAGE_NAME
  4. Dockerize your application on EC2
    Basically the same steps as on the mac.
    It might help to run the following (but be careful about which files you use and make sure that they are at the correct depth)
    git clone https://github.com/tzaffi/my-mashapes.git
